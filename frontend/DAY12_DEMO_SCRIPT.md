# Day 12 前端演示脚本

> 适用版本：前端闭环验收版  
> 后端地址：`http://127.0.0.1:8000`  
> 前端地址：`http://localhost:5173`

## 1. 启动

后端：

```bash
cd /Users/gjt/Documents/6.22
.venv/bin/python -m uvicorn backend.app.main:app --host 127.0.0.1 --port 8000
```

前端：

```bash
cd /Users/gjt/Documents/6.22/frontend
npm run dev
```

## 2. 首页闭环

1. 打开 `http://localhost:5173/`。
2. 确认顶部健康状态、最新观测时间和历史快照标识可见。
3. 确认地图工作区只显示已审核坐标点。
4. 点击一个散点，确认右侧抽屉展示站名、站码、水位、风险、趋势、观测时间和坐标状态。
5. 点击“查看历史曲线”，进入点位详情页。
6. 返回首页，点击“查看研判”，进入 `/assessments?stationCode=...`。

## 3. 点位详情

1. 在 `/points/:stationCode` 确认当前水位、观测时间、风险和趋势正常展示。
2. 确认已审核坐标显示“已审核”，并展示经纬度、来源和质量。
3. 切换历史曲线档位：最近 100 条、最近 500 条、最近 1000 条、最近 5000 条。
4. 若接口失败或点位不存在，确认页面显示友好错误，不白屏。

## 4. 数据状态

1. 打开 `/data-status`。
2. 确认积涝点数据范围显示最早时间、最新时间和记录数。
3. 确认最近导入批次显示导入时间、导入记录数和总行数。
4. 确认显示历史快照说明。
5. 确认水库数据仅作为状态摘要，不进入地图和积水风险主线。

## 5. 坐标校核

1. 打开 `/locations`。
2. 选择左侧待校核点位。
3. 若无候选，点击“生成高德候选坐标”。
4. 若后端未配置 `AMAP_WEB_SERVICE_KEY`，确认显示“高德服务未配置，请联系管理员”。
5. 对已有候选执行“通过”或“拒绝”。
6. 审核通过后返回首页和点位详情，确认坐标状态一致显示为已审核。

## 6. 规则研判

1. 打开 `/assessments`。
2. 确认表格展示全市站点、当前水位、风险、趋势、规则说明和生成时间。
3. 分别测试风险筛选和趋势筛选。
4. 组合测试“危险/警戒/关注 × 上升/下降/稳定/波动”等筛选。
5. 从首页抽屉跳转 `/assessments?stationCode=...`，确认目标行高亮并滚动到可见区域。

## 7. 演示边界说明

- 当前是历史快照数据，不是实时推送。
- 当前只做规则研判和趋势提示，不做模型预测。
- 前端不持有高德 Key，不接入真实高德 JS SDK。
- 水库水位只作为数据状态摘要，不参与地图、风险主线或历史曲线。
