# Day 1 一期/二期目标清单

> 日期：2026-06-23  
> 对应排期：Day 1 需求与范围确认  
> 当日交付物：一期/二期目标清单  
> 项目目标：建设深圳市积水实时监测地图系统，在 25 天内优先完成主线可运行闭环；预测模型属于主线支撑能力，告警中心、用户角色审计、工单等作为后置支线预留。

## 1. Day 1 结论

本项目不是只做一个地图页面，而是建设面向城市防汛和应急指挥的积水监测系统。25 天内按“先主线闭环、后分支扩展”的节奏推进：

```text
主线：实时水位接入 -> SQLite 入库 -> 坐标校核 -> 风险计算 -> 轻量预测模型 -> FastAPI 接口 -> 高德地图展示 -> 统计与历史查询 -> 数据状态
后置分支：PostGIS -> 调度任务 -> 点位级阈值 -> 扩展图层 -> 告警 -> 权限审计 -> 工单
```

主线必须先跑通。PostGIS、调度、点位级阈值和扩展图层可以在主线稳定后逐步补强；告警中心、用户角色审计和工单不作为当前优先完成目标。预测模型属于主线，但先采用规则型或轻量预测，不要求复杂机器学习训练。

## 2. 项目定位

系统服务对象是城市防汛和应急指挥人员，核心价值是快速回答：

- 哪里出现积水或积水风险。
- 当前风险等级是什么。
- 水位是否上升。
- 风险点集中在哪些行政区。
- 数据是否及时、可信、可追溯。
- 是否可以通过预测模型辅助研判未来积水风险。
- 后续是否需要生成告警或处置工单。

## 3. 一期目标

一期目标是完成基础可运行闭环。

### 3.1 一期必须完成

- 开放数据采集和原始数据归档。
- SQLite 入库和数据质量检查。
- 测站和积涝点基础信息整理。
- 高德地理编码候选坐标生成。
- 人工审核点位坐标。
- 统一风险等级计算。
- 简单趋势指标计算。
- 轻量预测模型或规则型预测结果展示。
- 高德地图展示积涝点、行政区边界和点位详情。
- 全市风险统计。
- 行政区风险统计。
- 单点历史水位查询。
- 数据导入状态查看。

### 3.2 一期验收标准

- 可以从本地数据或开放数据接口获得水位数据。
- 可以将数据导入 SQLite。
- 地图只展示坐标审核通过的点位。
- 点位 Marker 能按风险等级显示不同颜色。
- 点位详情能展示名称、水位、风险等级、趋势、观测时间。
- 预测结果能展示未来风险等级或趋势，并标明预测口径。
- 历史查询能展示单个点位的水位变化。
- 数据状态能展示导入批次、成功失败状态和更新时间。

## 4. 后置分支目标

后置分支目标是在主线闭环稳定后，逐步扩展成防汛指挥增强版本。当前阶段只需要保留设计位置和扩展接口思路，不要求先完成告警中心、用户角色审计和工单。

### 4.1 优先级较高的后续增强

- SQLite 迁移到 PostgreSQL + PostGIS。
- 采集脚本升级为后端调度任务。
- 支持点位级风险阈值。
- 接入降雨数据。
- 接入道路图层。
- 接入历史易涝点图层。

### 4.2 暂不优先完成的分支能力

- 增加站内告警中心或告警推送预留。
- 增加用户、角色和审计日志。
- 增加处置工单。

### 4.3 后置分支验收口径

- PostGIS 中能存储点位、行政区、道路、易涝点等空间数据。
- 后端能手动或定时触发数据同步任务。
- 不同点位可以配置不同风险阈值。
- 地图可以打开或关闭降雨、道路、历史易涝点图层。
- 告警、用户角色审计和工单先保留设计，不作为主线验收条件。

## 5. 技术路线

| 模块 | 技术选择 | 说明 |
|---|---|---|
| 后端 | FastAPI | 优先提供地图、统计、历史、校核、数据状态和预测等主线 API；告警、工单等 API 后置 |
| 前端 | Vue | 优先实现指挥地图、历史查询、点位校核、数据状态页面 |
| 地图 | 高德 JS API | 一期默认地图底座，封装 MapProvider 适配层 |
| 一期数据库 | SQLite | 快速完成数据验证和一期闭环 |
| 二期数据库 | PostgreSQL + PostGIS | 支持空间查询、行政区、道路、易涝点和图层扩展 |
| 数据采集 | Python 脚本 + 后端调度 | 先复用脚本完成主线，后续再纳入后端任务管理 |

## 6. 数据边界

### 6.1 已有数据

- 市水务局测站基本信息表。
- 市水务局积涝点水位数据。
- 市水务局水库水位表。
- 本地 SQLite 数据库。
- 已下载 CSV 和原始分页 JSON。

### 6.2 数据缺口

- 测站基础信息表缺少经纬度。
- 降雨数据源尚未正式接入。
- 道路数据和历史易涝点数据尚未正式接入。
- 复杂预测模型缺少训练数据和可靠评估口径，主线先采用规则型或轻量预测。

### 6.3 数据质量口径

- 空白水位保留在原始层，业务分析排除 `water_level_m IS NULL`。
- 完全重复 ID 记录去重后进入业务统计。
- 地图点位不能编造坐标，必须来自高德候选坐标审核或可信来源。
- 月份统计使用 `observed_at` 的真实日期，不只依赖文件所在月份目录。
