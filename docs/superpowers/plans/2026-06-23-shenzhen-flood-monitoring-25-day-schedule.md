## 4. 模块工作拆分

### 4.1 数据模块

目标：保证系统有可追溯、可查询、可计算的数据底座。

任务：

- 复核开放数据接口和字段。
- 保留原始 JSON 和 CSV。
- 整理 SQLite 表结构。
- 导入积涝点水位数据。
- 生成最新点位状态。
- 计算风险等级和趋势。
- 记录导入批次和异常信息。

验收：

- 可以重复执行数据导入。
- 可以查询某点位历史水位。
- 可以查询所有点位最新状态。
- 可以看到每次导入的成功或失败状态。

### 4.2 后端模块

目标：为前端提供稳定、清晰的业务 API。

任务：

- 搭建 FastAPI。
- 封装数据库访问。
- 实现地图点位接口。
- 实现全市和行政区统计接口。
- 实现历史查询接口。
- 实现坐标候选和审核接口。
- 实现数据状态接口。

验收：

- API 可以通过浏览器或接口工具访问。
- 关键接口返回结构稳定。
- 缺少数据时返回明确的空状态或错误信息。

### 4.3 地图与前端模块

目标：完成面向防汛指挥场景的可视化页面。

任务：

- 搭建 Vue 项目。
- 封装高德地图 Provider。
- 展示深圳行政区边界。
- 展示积涝点风险 Marker。
- 实现风险筛选、点位搜索、点位详情弹窗。
- 实现全市风险统计和高风险列表。
- 实现历史水位曲线。
- 实现点位坐标校核页面。
- 实现数据状态页面。

验收：

- 打开首页即可看到深圳地图和点位态势。
- 点位颜色能体现风险等级。
- 点击点位能看到水位、风险、趋势和观测时间。
- 可以查看单点历史曲线。
- 可以审核候选坐标。

### 4.4 文档与交付模块

## 6. 风险与缓冲

| 风险 | 影响 | 应对策略 |
|---|---|---|
| 测站基础信息缺少经纬度 | 地图无法真实落点 | 使用高德地理编码生成候选坐标，人工审核后再展示 |
| 开放数据接口不稳定或 appKey 权限不足 | 数据无法实时更新 | 保留本地 CSV/JSON 样本，演示时支持本地数据导入 |
| 行政区 GeoJSON 获取或坐标系不一致 | 区域边界展示异常 | 优先使用可信 GeoJSON，并统一转换到高德适配坐标 |
| 高德地图 Key 或配额问题 | 地图无法加载 | 提前申请 Key，并准备错误提示和截图备用 |
| 25 天范围较紧 | 功能实现不完整 | 坚持一期边界，降雨、道路、权限、复杂预测放二期 |
| 周末工作安排不稳定 | 进度波动 | 将周末任务设为相对独立模块，必要时顺延到下一个工作日 |

## 8. 优先级原则

如果时间紧张，优先保证以下主链路：

```text
水位数据导入 -> 点位坐标审核 -> 风险等级计算 -> 高德地图展示 -> 点位详情 -> 历史查询
```

可以压缩或后置：

- 多角色权限。
- 点位级阈值精细配置。
- 多点历史曲线对比。
- 降雨图层。
- 道路图层。
- 复杂预测模型。
- 大屏动效和视觉装饰。
