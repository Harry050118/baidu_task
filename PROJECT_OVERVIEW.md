# 深圳市积水监测与预测系统工程总览

本文档说明深圳市积水监测与预测系统的整体工程安排。当前项目不是单纯地图展示页，而是围绕“实时水位接入、原始数据归档、数据治理、坐标补全、地图监测、规则研判、训练数据沉淀、模型预测服务”建设一套可运行、可演示、可扩展的系统。

## 1. 项目目标

本项目目标是建设一套深圳市积水监测与预测系统，实时或定时接入水位监测数据，在地图上展示全市积水态势，并在历史数据沉淀后训练时序模型，辅助判断未来短时积水风险。

系统需要回答以下核心问题：

- 哪里出现积水或存在积水风险。
- 当前水位、风险等级和趋势是什么。
- 数据是否及时、可信、可追溯。
- 测站坐标来自哪里，精度和审核状态如何。
- 没有降雨数据时，能否先基于历史水位做短时趋势研判。
- 历史样本足够后，如何训练模型、评估模型并把预测结果接入系统。

当前阶段先完成监测闭环和规则研判；后续在历史样本、坐标特征和扩展数据逐步完善后，训练并接入时序预测模型。

## 2. 工程主线

项目主线如下：

```text
实时/定时水位数据接入
-> 原始 JSON/CSV 归档
-> SQLite 入库
-> 数据质量检查
-> 高德 API 初步坐标补全
-> 坐标来源、精度和审核状态记录
-> 规则研判和趋势提示
-> FastAPI 主线接口
-> Vue + 高德地图展示
-> 训练样本沉淀
-> 时序模型训练与评估
-> 模型预测服务接入
```

主线交付重点：

- 能从开放数据或本地归档数据获取水位数据。
- 能保存原始数据，保证数据可追溯。
- 能将水位数据导入本地 SQLite。
- 能处理空水位、重复记录、时间口径和导入批次状态。
- 能在缺少政府标准坐标时，用高德 API 生成初步坐标并标记来源和精度。
- 能计算实时风险等级、趋势提示和规则说明。
- 能通过 FastAPI 提供地图、统计、历史、坐标、研判/预测和数据状态接口。
- 能通过 Vue + 高德地图展示全市积水态势、坐标状态、规则研判和后期模型预测结果。

## 3. 主线模块

### 3.1 数据接入与归档

数据接入模块负责从深圳开放数据平台或本地归档文件获取水位数据。

主要输入：

- 市水务局积涝点水位数据。
- 市水务局测站基本信息表。
- 已下载 CSV 和原始分页 JSON。

主要输出：

- 原始分页 JSON。
- 汇总 CSV。
- 下载元数据。
- 导入批次状态。

工程要求：

- 原始数据必须保留在 `download/`。
- 不只保留清洗后的结果。
- 每次下载和导入都要能追溯来源、时间范围、行数和失败原因。

### 3.2 SQLite 数据底座

当前阶段优先使用 SQLite 完成主线闭环。

SQLite 的作用：

- 快速验证水位数据导入。
- 支撑风险等级和趋势计算。
- 支撑地图点位、统计、历史查询和数据状态接口。
- 保存临时坐标、审核状态、规则研判结果和后期预测结果。
- 降低早期环境复杂度。

本地数据库位置：

```text
data/local/shenzhen_water.db
```

后续可在主线稳定后迁移到 PostgreSQL + PostGIS，以增强空间查询和图层能力。

### 3.3 数据质量检查

数据质量检查用于保证地图、统计和训练样本可信。

当前重点口径：

- 空水位保留在原始层和历史明细中，但不参与风险计算和训练标签。
- 完全重复 ID 不重复进入业务统计。
- 月份统计应使用 `observed_at` 的真实日期。
- 导入批次需要记录成功、失败、行数和错误信息。
- 数据状态页面需要展示数据是否及时、是否导入成功。

### 3.4 坐标补全与校核

当前测站基础信息缺少经纬度，因此地图展示和模型空间特征不能编造坐标。

坐标策略：

```text
测站名称/地址/编码
-> 高德 API 初步坐标
-> 记录 coord_source=amap、coord_quality=provisional
-> 人工审核
-> 同步申请政府标准坐标
-> 标准坐标到位后校正或替换
-> 更新样本并重新评估/重训模型
```

坐标要求：

- 高德初步坐标可以用于初期地图落点和初期训练样本。
- 高德坐标必须保留来源、精度和审核状态。
- 不能把高德坐标等同于政府标准坐标。
- 政府标准坐标到位后，需要重新生成空间特征并复评模型。

### 3.5 规则研判与趋势提示

初期没有训练模型，因此不称为模型预测。该阶段只做规则研判和趋势提示。

规则研判包括：

- 风险等级：基于统一阈值计算正常、关注、警戒、危险、无数据。
- 趋势指标：基于最近有效水位序列计算上升、下降、稳定、波动、无数据。
- 数据新鲜度：判断数据是否延迟、缺失或正常。
- 规则说明：说明风险等级和趋势提示来自哪条规则。

规则研判输出：

- 点位 ID。
- 观测时间。
- 风险等级。
- 趋势提示。
- 规则版本。
- 规则说明。
- 生成时间。

### 3.6 时序模型训练与预测

后期在历史数据沉淀足够后，训练时序模型。

当前积涝水位数据已经覆盖 2026 年 1 月至 6 月。后续需要在现有积涝水位时序数据基础上接入降水量维度，把降水作为外生特征，用于评估“小雨、中雨、暴雨”等降雨等级下的积水风险预测能力。

训练和实时分析的基础不是日级汇总，而是 `download/市水务局积涝水位数据/` 中的近实时积涝点水位序列。当前数据字段表示水位，不直接表示积涝体积或积涝量；因此模型应从相邻观测中派生秒级水位变化量、变化速度和短时风险状态。一天粒度只适合做回看报表，不能作为实时防汛研判的主粒度。

在降水量数据尚未获取前，第一版先保留无降水基线模型；降水数据到位后，同步训练加入降水特征的增强模型。模型定位为：

```text
基线阶段：基于历史水位的短时水位趋势/风险预测
增强阶段：接入降水量后的多源积水风险等级预测
```

模型输入：

- 历史水位序列。
- 相邻观测的水位变化量。
- 按实际时间差计算的每秒水位变化速度。
- 时间特征。
- 测站特征。
- 高德临时坐标及坐标来源。
- 后续政府标准坐标。
- 降水量特征，获取后接入，包括小时雨量、累计雨量、降雨等级和站点/区域匹配关系。

预测目标和评估口径：

- 对 6 月样本做滚动测试：以 6 月 1 日为例，只使用 6 月 1 日之前的数据训练，6 月 1 日及之后作为测试；以 6 月 2 日为例，只使用 6 月 2 日之前的数据训练，6 月 2 日及之后作为测试，依此类推。
- 对 6 月数据按降雨等级划分场景，至少区分小雨、中雨、暴雨，并分别统计预测准确率。
- 地图或结果页展示降雨/风险等级时，颜色由浅到深表达等级增强。

注意：时序模型不能随机切分，应按预测目标时间划分训练集、验证集和测试集；滚动测试中不得使用预测日及之后的数据训练模型，避免时间泄漏。

### 3.7 后端主线接口

后端采用 FastAPI。

主线 API 包括：

| API | 用途 |
|---|---|
| `GET /api/map/points` | 地图点位、最新水位、风险等级、趋势、坐标来源 |
| `GET /api/map/districts` | 深圳行政区边界 |
| `GET /api/stats/overview` | 全市风险统计 |
| `GET /api/stats/districts` | 行政区风险统计 |
| `GET /api/points/{id}/history` | 单点历史水位 |
| `POST /api/locations/geocode-candidates` | 高德候选坐标 |
| `POST /api/locations/{id}/review` | 审核点位坐标 |
| `GET /api/assessments` | 查询规则研判结果 |
| `GET /api/predictions` | 查询模型阶段预测结果 |
| `GET /api/imports/latest` | 最近导入批次和数据状态 |

### 3.8 前端主线页面

前端采用 Vue，地图或地理编码能力通过后端接口接入高德 Web 服务。真实高德 Key 只保留在后端 `.env`，前端不直接持有高德 Key。

主线页面包括：

| 页面 | 作用 |
|---|---|
| 指挥地图首页 | 展示积水点位、风险颜色、行政区边界、态势统计 |
| 历史查询页 | 查询单点历史水位和水位变化 |
| 点位校核页 | 审核高德候选坐标，确认地图落点 |
| 数据状态页 | 展示数据源、导入批次、成功失败状态和错误信息 |
| 研判/预测结果页 | 初期展示规则研判，后期展示模型预测结果 |

地图弹窗应展示：

- 点位名称。
- 当前水位。
- 风险等级。
- 趋势提示。
- 观测时间。
- 坐标来源和审核状态。
- 历史曲线入口。
- 后期模型预测摘要。

## 5. 技术架构

当前技术路线：

| 层级 | 技术或目录 | 说明 |
|---|---|---|
| 数据采集 | Python 脚本 | 下载开放数据、归档 JSON/CSV |
| 数据存储 | SQLite | 当前主线数据库 |
| 坐标补全 | 高德 API + 人工审核 | 初步补点位坐标，保留来源和精度 |
| 数据治理 | Python/FastAPI services | 质量检查、风险等级、趋势提示 |
| 后端服务 | FastAPI | 提供地图、统计、历史、坐标、研判/预测和数据状态接口 |
| 前端应用 | Vue | 展示指挥地图和后台主线页面 |
| 地图和坐标能力 | 后端高德 Web 服务接口 | 候选坐标、地理编码和后续地图能力，不在前端暴露真实 Key |
| 模型训练 | Python | 后期训练时序模型和评估模型 |
| 后续空间能力 | PostgreSQL + PostGIS | 后续支持空间查询和扩展图层 |

系统分层：

```text
数据源层
-> 数据采集与归档层
-> SQLite 数据底座
-> 坐标补全与数据治理层
-> 规则研判层
-> FastAPI 服务层
-> Vue + 高德地图应用层
-> 后期模型训练与预测服务层
```
